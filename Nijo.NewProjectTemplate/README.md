# Nijo Application Builder - New Project Template

これは Nijo Application Builder を用いて作成される新しいプロジェクトのテンプレートです。
このテンプレートには、独立してデバッグ可能な必要最低限の構成のみが含まれ、アプリケーションのユーザーの目線で必要な機能や、開発生産性を上げるためのツールなどは含まれていません。

## 動作環境

このテンプレートのデバッグを実行するには以下のツールがインストールされ、パスが通っている必要があります。

- **dotnet** : 9.0.302 以上
- **npm** : 10.8.2 以上

## フォルダ構造

フォルダ名は nijo.xml で変更することができます。

- **client** : Webクライアント側のプロジェクト。既定では Vite + React + TypeScript で構築されています。自動生成されるソースを活用するためには、このうち TypeScript のみが必須で、ほかのライブラリは使用しなくても構いません。Webを使用しない場合、このプロジェクトを削除しても構いません。
- **core.AutoGenerated** : .NET側の自動生成される主なソースが格納されるプロジェクト。純粋なクラスライブラリで、実行基盤を問いません。
- **core** : core.AutoGenerated を利用し、自動生成されない処理を記述するプロジェクト。純粋なクラスライブラリ。 core.AutoGenerated と分離されている理由は、自動生成されるプロジェクトは変更頻度が低い割にビルドに時間がかかるため。
- **webapi** : ASP.NET Core Web API プロジェクト。coreを利用してサーバー側処理を実行します。ここに生成されるHTTPリクエスト・レスポンスの型はclientに生成されるソースと常に同期されていることが保証されています。Webを使わない場合、このプロジェクトを削除しても構いません。

## 参考: このテンプレートを作成した手順

- **dotnet** : 9.0.302 以上
- **npm** : 10.8.2 以上

```ps1
cd <このREADME.mdがあるフォルダ>

# .NET の各種プロジェクトを新規作成する (dotnet ver.9.0.302)
dotnet new classlib -o core
dotnet new classlib -o core.AutoGenerated
dotnet new webapi -o webapi

Rename-Item -Path "core\core.csporj"                             -NewName "MyApp.csproj"
Rename-Item -Path "core.AutoGenerated\core.AutoGenerated.csporj" -NewName "MyApp.AutoGenerated.csproj"
Rename-Item -Path "webapi\webapi.csporj"                         -NewName "MyApp.WebApi.csproj"

# 上記3プロジェクトを含むソリューションを作成
dotnet new sln --name MyApp
dotnet sln add --in-root core\MyApp.csproj
dotnet sln add --in-root core.AutoGenerated\MyApp.AutoGenerated.csproj
dotnet sln add --in-root webapi\MyApp.WebApi.csproj

# Vite + React + TypeScript の新規プロジェクトを作成する (npm ver.10.8.2)
npm create vite@7.1.2 my-app -- --template react-ts
Rename-Item -Path "my-app" -NewName "client"

# 以下は手作業で作成して配置
# .editorconfig
# .gitignore
# .gitattributes
```
