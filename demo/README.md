# Nijo Application Builder デモプロジェクト

## 番号

- 000～: 基本的な説明
- 100～: 実用的な例
- 200～: エッジケース。主に動作確認目的

## デモ新規作成時

1. `nijo new <プロジェクト名>` でプロジェクトを作成
2. 不要なファイルを削除
   - README.md
   - package.json の依存パッケージ
3. [VSCode設定ファイル](../nijo.code-workspace) にフォルダ名を追加。VSCode > F1 > Run Task からのデバッグコマンド等で新規追加プロジェクトを選択できるようにする
4. [モノレポのルートのpackage.json](../package.json) に新規追加プロジェクトのclientフォルダを追加

## デモ編集時

### そのデモ独自のコードの場合

デモプロジェクトのソースを直接編集する。

### デバッグコマンドや集約定義に依存しない設定など

デモではなく [テンプレプロジェクト側](../Nijo.NewProjectTemplate/) を編集する。デモで見つかった不具合をプロダクトコードにフィードバックするため。

テンプレプロジェクト側を編集後、 [ソース最新化.bat](./ソース最新化.bat) で最新化する。 VSCode の Run Task からこれを実行する形でも可。

### npmパッケージの新規追加の場合

基本的に各プロジェクトのpackage.jsonではなくモノレポのルートのpackage.jsonに追加すること。
node_modulesの容量削減のため。
