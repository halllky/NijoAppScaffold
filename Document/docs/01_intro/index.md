# はじめに

Nijoは、業務システム（エンタープライズアプリケーション）のスクラッチ開発を効率化するコード自動生成ツールです。
アプリケーションのデータ構造を「スキーマ」として一度だけ定義し、そこから画面からデータベースまでの定型的なコードを自動生成します。

## とにかく動かしてみたい方へ

[デモ](./01_demo.md) では、ローカル環境でデモサイトを動かし、Nijoを実際に体験する手順を説明しています。

## 特徴

業務システム開発で最も困難なのは、複雑なデータ構造と業務ルールの整合性をアプリケーション全体で保ち続けること、そして関係者間で各データの意味や役割の認識を合わせ続けることです。
Nijoはこの課題を次の3つのしくみで解決します。

### 1. GUIエディタによる設計の可視化

専用のGUIエディタ（`nijo serve`）で、データ構造やリレーションをER図のようなグラフとして可視化します。
「このデータが消えたら、関連するこのデータはどうなる？」「この画面で検索されるデータの構造は？」といったシステムの挙動を、コードを書く前にチームで確認し、実装方針の合意形成を行えます。

編集内容は `nijo.xml` というXMLファイルに保存されます。これが設計ドキュメントであると同時に、コード生成の元データになります。

![GUIエディタ画面](/img/gui-sample-2.png)

### 2. モデルによるスキーマ定義

Nijoでは、アプリケーションを「モデル」という単位で定義します。主要なモデルは次の3つです。

- **データモデル**（上の画像のオレンジ色の図形）: 永続化されるデータの形。同一トランザクションで更新されるべき複数のテーブルを1つの塊として扱います。
- **クエリモデル**（緑色の図形）: エンドユーザーが検索・一覧表示する粒度のデータの形。
- **コマンドモデル**（青色の図形）: 更新処理、画面初期表示、帳票出力など、入力を受けて結果を返す処理。

この厳格なモデリングにより、データの不整合を防ぎ、保守性の高いシステム設計を促します。
各モデルの詳細は [モデリングの基礎](../02_workflow/020000_modeling.md) を参照してください。

### 3. フルスタックかつ型安全なコード生成

定義されたモデルから、Entity Framework Core のテーブル定義、C#のクラスとCRUD処理、ASP.NET Core Web API、TypeScriptの型定義とAPIクライアントまでを一気通貫で生成します。
フロントエンドからデータベースまで型定義が同期されるため、変更に強い開発が可能です。

## 技術スタック

Nijoが生成するアプリケーションは、以下の技術スタックを採用しています。

- **データベース**: 任意のRDBMS（EF Coreがサポートするもの）。デフォルトテンプレートはSQLite
- **バックエンド**: .NET 9 以上 / ASP.NET Core / Entity Framework Core
- **フロントエンド**: TypeScript。JavaScriptフレームワークやCSSライブラリの採用は任意
