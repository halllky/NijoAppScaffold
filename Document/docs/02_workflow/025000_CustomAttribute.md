# カスタム属性（CustomAttributes）

カスタム属性は、スキーマ定義のXML要素に任意のメタデータを付与するための機能です。
自動生成後のアプリケーションで**横断的処理**（ロギング、バリデーション、UI制御など）を実装する際に、リフレクションやメタデータ参照でアクセスできます。

## 概要

カスタム属性を使うと以下が自動生成されます。

- **C# Attributeクラス**（`NijoAttr.MaskingAttribute` など）— リフレクションで利用可能
- **TypeScript型定義とメタデータ値**（`MetadataForPage` の各プロパティ）— パス名でアクセス可能
- **バリデーションフック**（`IsValidation="True"` の場合）— `ApplicationService` の abstract メソッド

## nijo.xml での定義

カスタム属性は `<CustomAttributes>` セクション内に定義します。

```xml
<NijoAppScaffold>
  <!-- ... 集約定義 ... -->

  <CustomAttributes>
    <!--ログ出力時に値をそのまま出さずにマスキングさせる。パスワードなどに指定する。-->
    <Custom-4d868e55-06d6-44eb-aa97-901f39dd1782
      AvailableModels="data-model,query-model,command-model,structure-model,enum,value-object"
      DisplayName="マスキング"
      PhysicalName="Masking"
      Type="Boolean" />
  </CustomAttributes>
</NijoAppScaffold>
```

### 属性一覧

| 属性名 | 必須 | 説明 |
|---|---|---|
| `PhysicalName` | ○ | C# Attributeクラス名・TypeScriptプロパティ名の基となる物理名（PascalCase） |
| `DisplayName` | ○ | 表示名 |
| `Type` | ○ | 値の型。`Boolean` / `String` / `Decimal` / `Enum` |
| `AvailableModels` | | 付与を許可するモデルの種類（カンマ区切り）。未指定の場合は全モデルで使用可能 |
| `IsValidation` | | `True` を設定すると、ApplicationService にバリデーションフックが生成される |
| `Comment` | | ドキュメント用コメント |

### 型（Type）の違い

| Type | XML上の値例 | C#の型 | TypeScriptの型 |
|---|---|---|---|
| `Boolean` | `"True"` | `bool` | `boolean` |
| `String` | `"任意文字列"` | `string?` | `string` |
| `Decimal` | `"100"` | `decimal?` | `number` |
| `Enum` | `"OptionA"` | 内部 enum | ユニオン型 |

## 集約メンバーへの付与

定義したカスタム属性は、スキーマXML上でその UniqueId を属性名として付与します。

```xml
<ユーザー Type="data-model">
  <!-- Custom-4d868e55... が「マスキング」属性の UniqueId -->
  <パスワード
    Type="word"
    Custom-4d868e55-06d6-44eb-aa97-901f39dd1782="True" />
</ユーザー>
```

スキーマエディタ（Web UI）ではプロパティパネルから操作できます。

## 生成されるコード

### C# Attributeクラス（NijoAttr サブ名前空間）

```csharp
// NijoAttr サブ名前空間に生成される
namespace MyApp.NijoAttr;

[AttributeUsage(AttributeTargets.All, Inherited = false, AllowMultiple = false)]
public sealed class MaskingAttribute : System.Attribute {
    public MaskingAttribute() { }
}
```

付与対象クラスの DbEntity プロパティにも自動で `[NijoAttr.Masking]` が付きます。

### TypeScript メタデータ

```typescript
// __autoGenerated/util/metadata-for-page.ts に生成される

export namespace MetadataForPage {
  export type StructureMetadata = {
    // ...
    /** マスキング */
    masking?: boolean
  }
  export type ValueMetadata = {
    // ...
    /** マスキング */
    masking?: boolean
  }
}
```

### バリデーションフック（IsValidation="True" の場合）

```csharp
// ApplicationService に abstract メソッドが生成される

/// <summary>
/// カスタム属性 'NotNegative' 用の入力検証フック。
/// null を返すとバリデーションOK、文字列を返すとその内容でエラー登録されます。
/// </summary>
public abstract string? ValidateNotNegative(int? value, System.Reflection.PropertyInfo propertyInfo);
```

## 利用例

### C# — リフレクションによるログマスキング

```csharp
// Core/Logging.cs
using MyApp.NijoAttr;

private sealed class MaskingTypeResolver : DefaultJsonTypeInfoResolver {
    public override JsonTypeInfo GetTypeInfo(Type type, JsonSerializerOptions options) {
        var typeInfo = base.GetTypeInfo(type, options);

        if (typeInfo.Kind == JsonTypeInfoKind.Object) {
            foreach (var prop in typeInfo.Properties) {
                var provider = prop.AttributeProvider;
                if (provider == null) continue;

                // MaskingAttribute が付いているプロパティを *** に置換する
                if (provider.IsDefined(typeof(MaskingAttribute), inherit: true)) {
                    prop.CustomConverter = GetMaskingConverter(prop.PropertyType);
                }
            }
        }
        return typeInfo;
    }
}
```

### TypeScript — メタデータパスによるUI制御

```typescript
// client/src/input/Field.tsx
import { MetadataForPage } from "../__autoGenerated/util/metadata-for-page"

// 集約名（PhysicalName）でルート集約のメタデータを取得
const allMetadata = MetadataForPage.getAll()
const metadata = allMetadata['ユーザー']

// メンバー名で個別フィールドのメタデータを取得
const fieldMetadata = metadata.members['パスワード']

// カスタム属性の値を参照
if ('masking' in fieldMetadata && fieldMetadata.masking) {
  // パスワードフィールドの場合はマスク入力にする
}
```

子孫集約のメタデータはメンバー名を順にたどることでアクセスできます。

```typescript
// 売上.明細[].商品名 のメタデータ
const memberMetadata = allMetadata['売上'].members['明細'].members['商品名']
```

### TypeScript — 数値表示のカスタマイズ

```typescript
// IsCurrency, IsQuantity による表示フォーマット切り替え
if (fieldMetadata.isCurrency) {
  suffix = "円"
  commaSeparated = true
} else if (fieldMetadata.isQuantity) {
  suffix = "個"
  commaSeparated = true
}
```

### C# — バリデーションフックの実装

```csharp
// ApplicationService のサブクラスで実装する
public override string? ValidateNotNegative(int? value, PropertyInfo propertyInfo) {
    if (value < 0) return "0以上の値を入力してください。";
    return null;
}
```

## デモプロジェクトでの定義例

`demo/101_販売管理システム` では以下のカスタム属性が定義されています。

| PhysicalName | Type | IsValidation | 用途 |
|---|---|---|---|
| `Masking` | Boolean | — | ログ出力時にパスワード等をマスキング |
| `IsCurrency` | Boolean | — | 金額項目として「円」サフィックス・3桁カンマ区切り表示 |
| `IsQuantity` | Boolean | — | 数量項目として「個」サフィックス・3桁カンマ区切り表示 |
| `CustomMaxLength` | Decimal | — | 画面入力の最大文字数（DB定義とは独立） |
| `NotNegative` | Boolean | True | 0未満の値を入力不可とするバリデーション |
